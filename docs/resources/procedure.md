# komodo_procedure (Resource)

Manages a Komodo procedure resource. A procedure consists of one or more ordered stages. Stages run sequentially; executions within a stage all run in parallel.

## Example Usage

```terraform
resource "komodo_procedure" "example" {
  name = "deploy-pipeline"

  # Stage 1: Deploy the stack
  stage {
    name = "Deploy"

    execution {
      type = "DeployStack"
      parameters = {
        id        = komodo_stack.app.id
        stop_time = "60"
      }
    }
  }

  # Stage 2: Run health checks in parallel
  stage {
    name = "Health Checks"

    execution {
      type = "RunAction"
      parameters = {
        id = komodo_action.health_check.id
      }
    }

    execution {
      type    = "RunBuild"
      enabled = false
      parameters = {
        id = komodo_build.app.id
      }
    }
  }

  # Stage 3: Notify via another procedure
  stage {
    name = "Notify"

    execution {
      type = "RunProcedure"
      parameters = {
        id = komodo_procedure.send_notification.id
      }
    }
  }

  schedule {
    format     = "Cron"
    expression = "0 2 * * *"
    enabled    = true
    timezone   = "America/New_York"
  }

  failure_alert_enabled = true
}
```

## Execution Types

The `type` attribute of an `execution` block selects which Komodo action to run. All parameters are passed as string values in the `parameters` map.

### Deployment Executions

| Type | Description | Required | Optional |
|------|-------------|----------|---------|
| `Deploy` | Deploys/redeploys the container (pull image, stop, start). | `id` | `stop_signal`, `stop_time` |
| `StartDeployment` | Starts the target deployment container. | `id` | — |
| `StopDeployment` | Stops the target deployment container. | `id` | `signal`, `time` |
| `RestartDeployment` | Restarts the target deployment container. | `id` | — |
| `DestroyDeployment` | Stops and removes the target deployment container. | `id` | `signal`, `time` |
| `PauseDeployment` | Pauses the target deployment container. | `id` | — |
| `UnpauseDeployment` | Unpauses the target deployment container. | `id` | — |
| `PullDeployment` | Pulls the latest image for the target deployment. | `id` | — |

```terraform
# Deploy (redeploy) a deployment
execution {
  type = "Deploy"
  parameters = {
    id          = komodo_deployment.app.id
    stop_signal = "SIGTERM"
    stop_time   = "30"
  }
}

# Stop a deployment
execution {
  type = "StopDeployment"
  parameters = {
    id   = komodo_deployment.app.id
    time = "10"
  }
}
```

### Stack Executions

| Type | Description | Required | Optional |
|------|-------------|----------|---------|
| `DeployStack` | `docker compose up` on the target stack. | `id` | `services`, `stop_time` |
| `DeployStackIfChanged` | `docker compose up` only if contents have changed. | `id` | `stop_time` |
| `DestroyStack` | `docker compose down` on the target stack. | `id` | `services`, `stop_time`, `remove_orphans` |
| `StartStack` | `docker compose start` on the target stack. | `id` | `services` |
| `StopStack` | `docker compose stop` on the target stack. | `id` | `services`, `stop_time` |
| `RestartStack` | `docker compose restart` on the target stack. | `id` | `services` |
| `PauseStack` | `docker compose pause` on the target stack. | `id` | `services` |
| `UnpauseStack` | `docker compose unpause` on the target stack. | `id` | `services` |
| `PullStack` | `docker compose pull` on the target stack. | `id` | `services` |
| `RunStackService` | `docker compose run` for a one-time service command. | `id`, `service` | `command`, `detach`, `entrypoint`, `env`, `no_deps`, `no_tty`, `pull`, `service_ports`, `user`, `workdir` |

```terraform
# Deploy a stack
execution {
  type = "DeployStack"
  parameters = {
    id        = komodo_stack.app.id
    stop_time = "60"
  }
}

# Deploy only if contents changed
execution {
  type = "DeployStackIfChanged"
  parameters = {
    id = komodo_stack.app.id
  }
}

# Destroy a stack
execution {
  type = "DestroyStack"
  parameters = {
    id             = komodo_stack.app.id
    remove_orphans = "true"
  }
}
```

### Server Prune Executions

All server prune types take only `id` (the server ID) as their parameter.

| Type | Description |
|------|-------------|
| `PruneDockerBuilders` | Prunes docker builders on the target server. |
| `PruneBuildx` | Prunes the docker buildx cache on the target server. |
| `PruneContainers` | Prunes docker containers on the target server. |
| `PruneImages` | Prunes docker images on the target server. |
| `PruneNetworks` | Prunes docker networks on the target server. |
| `PruneSystem` | Prunes the docker system (including volumes) on the target server. |
| `PruneVolumes` | Prunes docker volumes on the target server. |

```terraform
execution {
  type = "PruneImages"
  parameters = {
    id = komodo_server.main.id
  }
}
```
